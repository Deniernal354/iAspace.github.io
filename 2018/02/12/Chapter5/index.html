第五章 继承

类 超类 子类

定义子类

  1. 关键词extends表示继承
  2. 在Java中的所有继承都是公有继承
  3. 已存在的类被称为超类/父类，新类被称为子类
  4. 子类比超类拥有的功能更丰富
  5. 超类方法不能调用子类方法和域，反之则可以
  6. 设计程序时可将通用类放在超类中，特殊作用的方法放在子类中

覆盖方法

  1. 想要访问超类中的私有域，必须借助公共接口。借助关键词Super
  2. 子类可以增加域，方法或覆盖超类的方法，不能删除继承的任意域和方法

子类的构造器

  1. 使用Super(...)调用超类的构造器，对这部分私有域进行初始化。
  2. 使用Super调用构造器的语句必须是子类构造器的第一句。
  3. 如果子类没有显式调用超类的构造器，则自动调用默认(无参数)的构造器
  4. 一个对象类型可以指示多个实际类型的现象称为多态
  5. 在运行时能够自动选择调用哪个方法的现象称为动态绑定

继承层次

  1. 由一个公共超类派生出来的所有类的集合被称为继承层次，继承不限于一个层次
  2. 在继承层次中从某个特定的类到其祖先的路径被称为继承链

多态

  1. “is-a”规则，在子类的每个对象也是超类的对象
  2. “is-a”规则的另一种描述：置换法则，程序出现超类对象的任何地方都能替换为子类对象
  3. 在Java程序设计中对象变量是多态的。一个超类对象可以引用它的任何一个子类对象
  4. 不能将一个超类的引用赋给子类变量

方法调用

  1. 编译器查看对象的声明类型和方法名
  2. 编译器将查看调用方法时提供的参数类型，方法中存在一个与提供的参数完全匹配的方法，称为重载解析
  3. 如果是private、static、final方法或构造器，编译器将准确知道调用那种方法，这种调用方式被称为静态绑定
  4. 当程序采用动态绑定调用时，虚拟机将调用与引用方法实际类型最为接近的类的方法
    动态绑定特性:无需对现存的代码进行修改，就可以对程序进行扩展

阻止继承

  1. final的类和方法，不允许扩展的类被称为final类
  2. 定义类时使用final修饰符，这个类就是final类
  3. 类的特定方法也可以被声明为final，子类就不能覆盖这个方法
  4. 如果将一个类定义为final，则方法自动成为final
  5. 目的是确保他们不会在子类中改变语义

强制类型转换

  1. 将一个类型强制转换成另一个类型的过程称为类型转换
  2. 唯一原因:在暂时忽视对象的类型之后，使用对象的全部功能
  3. 将一个超类的引用赋给一个子类时，必须进行类型转换，反之则不用进行
  4. 在进行类型转换之前先用instanceof查看是否能够成功的进行转换
  5. 只能在继承层次内进行类型转换，在超类转换为子类之前，应使用instanceof进行检查

抽象类

  1. 在类的继承层次中，上层的类应该更具有通用性、更加抽象
  2. 使用关键词abstranct可以不用实现方法
  3. 包含一个或多个抽象方法的类，本身必须被声明为抽象的
  4. 抽象类可包含具体的数据和方法
  5. 类不含抽象方法也可定义抽象类
  6. 抽象类不能被实例化
  7. 可以定义一个抽象类的对象变量，但它只能引用非抽象子类的对象
扩展抽象类
  1. 在抽象类中定义部分抽象类方法或不定义，则子类也必须是抽象的
  2. 定义所有抽象方法，则子类不是抽象的
受保护访问

超类的某些方法被子类访问,或子类方法访问超类的某个域则需要protected

控制可见的四个访问修饰符

  1. 对本类可见 — private
  2. 对所有类可见 — public
  3. 对本包和所有子类可见 — protected
  4. 对本包可见 — 默认

Object：所有类的超类

  1. 在Java中每个类都是由它扩展来的
  2. 可以用Object类型的变量引用任何类型的对象
  3. Object类型的变量只能作用于各种值的通用持有者
  4. 在Java中只有基本类型不是对象，所有数组都扩展了Object

hashcode方法

  1. 散列码是由对象导出的一个整型值
  2. 散列码是没有规律的
  3. hashcode方法定义在Object类中，每个对象都有一个默认的散列码，其值为对象的存储地址

泛型数组列表

ArrayList是一个采用类型参数的泛型类

对象包装器和自动装箱

  1. 所有的基本数据类型都有一个与之对应的类，这些类被称为包装器
  2. 对象包装器是不可变的，不能定义子类，不能更改数值

反射

能够分析类能力的程序称为反射

  1. 在运行时分析类的能力
  2. 在运行时查看对象
  3. 实现通用的数组操作代码
  4. 利用Method对象
    …

继承的设计技巧

  1. 将公共操作和域放在超类中
  2. 不要使用受保护的类
  3. 使用继承实现“is-a”关系
  4. 除非所有继承的方法都是有意义的，否则不要用继承
  5. 再覆盖方法时，不要改变预期的行为
  6. 使用多态而非类型信息
  7. 不要过多使用反射
